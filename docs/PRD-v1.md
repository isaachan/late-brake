# Late Brake - 圈速数据分析工具 v1

## 1. 产品概述

Late Brake 是一个纯命令行（CLI）的赛车圈速数据分析工具，用于对比分析不同圈速数据文件中的单圈表现，帮助赛车手找出最佳走线和刹车时机。

产品设计为灵活、轻量化的纯CLI工具。**Late Brake**（晚刹车）这个名字体现了赛车运动中最具挑战性也是最关键的技术环节——找到刹车点的最优解。

## 2. 目标用户

- 业余赛车手/职业赛车手
- 赛车数据分析师
- 需要分析车载GPS圈速数据的爱好者

用户特点：
- 习惯使用命令行工具
- 需要快速对比不同圈速
- 数据来源多样（不同记录仪输出不同格式）
- 注重效率和灵活性

## 3. 核心功能

### 3.1 数据导入与格式转换
- **支持多种输入格式**：NMEA、VBO（AiM / VBox）等常见圈速数据格式
- **统一内部格式**：将不同源数据转换为标准格式保存，便于后续处理
- **批量导入**：支持一次性导入多个文件

### 3.2 自动圈速分割
- 根据起点/终点线自动识别单圈（Lap）
- 输出所有识别到的圈速列表
- 显示每圈的圈时、分段时间等基本信息
- 允许用户手动调整分割点（v1暂不实现）

### 3.3 任意圈速对比
- 支持对比任意两个Lap
- 两个Lap可以来自同一个数据文件，也可以来自不同文件
- 输出对比结果：圈时差、各个分段的时间差、速度差等

### 3.4 纯CLI单次命令模式
- 无REPL交互式会话，每次执行一个命令完成一个任务
- 命令式设计，简单直接，便于脚本化和自动化
- 支持单次执行完成任务后直接退出

### 3.5 多种输出格式
- 默认输出人类可读的文本格式到stdout
- 支持JSON格式输出，便于后续程序处理
- 所有命令都支持JSON输出选项

## 4. 交互流程

### 4.1 命令格式

所有操作都通过单次CLI命令完成，执行完毕后直接退出：

#### 4.1.1 加载并查看Lap信息
```bash
$ late-brake load xxx.nmea xxx.vbo
```
输出每个文件包含的Lap信息（圈数、每圈圈时）。

#### 4.1.2 对比两个指定Lap
```bash
$ late-brake compare xxx.nmea 1 xxx.vbo 3
```
对比 `xxx.nmea` 的第1圈 和 `xxx.vbo` 的第3圈，输出对比结果。

### 4.2 输出格式选项

默认输出人类可读的文本格式：

```
$ late-brake load data1.vbo data2.nmea

文件: data1.vbo
  共 3 圈
  Lap 1: 1:25.323
  Lap 2: 1:24.876
  Lap 3: 1:24.102
文件: data2.nmea
  共 2 圈
  Lap 1: 1:24.561
  Lap 2: 1:23.988
```

使用 `--json` 参数输出JSON格式：

```bash
$ late-brake load data1.vbo data2.nmea --json
```

```json
{
  "files": [
    {
      "path": "data1.vbo",
      "lapCount": 3,
      "laps": [
        {"number": 1, "time": 85.323},
        {"number": 2, "time": 84.876},
        {"number": 3, "time": 84.102}
      ]
    },
    {
      "path": "data2.nmea",
      "lapCount": 2,
      "laps": [
        {"number": 1, "time": 84.561},
        {"number": 2, "time": 83.988}
      ]
    }
  ]
}
```

对比命令同样支持JSON输出：

```bash
$ late-brake compare data1.vbo 3 data2.nmea 2 --json
```

## 5. 数据格式

Late Brake 将不同来源的圈速数据转换为统一的内部JSON数据结构，便于后续处理和分析。完整的数据结构定义请参考：  
[数据格式文档](./data-format.md)。

概述：
- **数据点结构**：每个采样点包含时间、GPS坐标、速度、距离，以及可选的G值、油门、刹车等传感器数据
- **单圈结构**：每个单圈包含圈速、起止位置、完整状态，以及所有数据点数组


## 6. 支持的输入文件格式
v1支持 NMEA 和 VBO 格式，后续将支持更多格式

| 格式 | 说明 | 状态 |
|------|------|------|
| NMEA | GPS标准输出 | v1支持 |
| VBO  | VBox格式 | v1支持 |
| CSV  | 通用CSV | 后续迭代 |
| AIM  | AiM记录仪格式 | 后续迭代 |


## 7. 输出格式规范

### 7.1 JSON输出要求

- 所有命令都支持 `--json` 参数输出JSON格式
- JSON输出到stdout，便于管道传递给其他程序
- 结构清晰，字段语义明确
- 支持机器解析，便于后续自动化处理

### 7.2 后续迭代方向

v1 之后考虑：

1. **可视化输出**：生成速度-距离曲线图数据，可以导入Excel或Python绘图
2. **分段计时对比**：支持自定义 sector 分段
3. **GPS轨迹绘制**：输出可查看的赛道地图轨迹对比
4. **最优圈自动识别**：自动找出最快圈
5. **刹车点分析**：专门分析对比两个圈的刹车点差异（产品核心诉求）
6. **更多格式支持**：添加更多记录仪格式支持
7. **平滑处理**：对GPS数据做滤波平滑，提高分析精度
